Git学习笔记之Rebase基础操作

场景

在开发新功能时，将其他开发者提交到master分支的代码合并到自己当前开发的分支。

操作

切换到master分支并拉取最新代码

将master分支的代码合并到自己当前开发的功能分支

开发后提交代码

心得

Rebase的流程如下

刚执行完(master) git checkout -b feature1时

执行完(feature1) git rebase master之后

Rebase可以理解为将更新后master的分支重新设定为feature1的“基础（起点）”。
